# Unit-тесты Урок 5



## Задание 1. 
Представьте, что вы работаете над разработкой простого приложения для записной книжки, которое позволяет пользователям добавлять, редактировать и удалять контакты.
Ваша задача - придумать как можно больше различных тестов (юнит-тесты, интеграционные тесты, сквозные тесты) для этого приложения. Напишите название каждого теста, его тип и краткое описание того, что этот тест проверяет.

test_UI_addContact_displayCorrectly

Тип: Интеграционный тест
Описание: Проверяет, что при добавлении контакта через пользовательский интерфейс, контакт корректно отображается в списке контактов.
test_contactLifecycle

Тип: Интеграционный тест
Описание: Проверяет полный жизненный цикл работы с контактом - создание, редактирование и удаление контакта из записной книжки.
Сквозные тесты:

test_dataPersistence

Тип: Сквозный тест
Описание: Проверяет сохранение данных контактов между различными сеансами использования приложения, убеждаясь, что данные не теряются после перезапуска приложения.
test_contactListSorting

Тип: Сквозный тест
Описание: Проверяет правильность сортировки списка контактов по заданному критерию (например, по алфавиту или по времени добавления).
